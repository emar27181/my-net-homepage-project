# clips.json スコア採点ガイド

## スコア項目一覧

| 項目 | スケール | 説明 |
|------|----------|------|
| `smooth` | 1〜5 | エイムや動きのスムーズさ |
| `rotation` | 1〜5 | ローテーション・立ち回りの質 |
| `clutch` | 1〜5 | クラッチ状況（相手人数） |
| `highlight` | 1〜5 | 映像としてのハイライト度 |

---

## clutch（クラッチ）採点ルール

相手の人数をそのままスコアにする。

| タイトルの表記 | clutch値 |
|---------------|---------|
| 記載なし（通常キル） | 1 |
| 1v2 Clutch | 2 |
| 1v3 Clutch | 3 |
| 1v4 Clutch | 4 |
| 1v5 Clutch | 5 |

---

## smooth / rotation / highlight（採点基準 TBD）

> **現在の値はすべて暫定値（3）。採点基準は後で定義予定。**

以下は検討中の基準案：

### smooth（エイムのスムーズさ）
| スコア | 目安 |
|--------|------|
| 5 | ワンタップ・フリック等、精度の高いキルが続く |
| 3 | 普通 |
| 1 | スプレー多用・エイムが荒い |

### rotation（立ち回りの質）
| スコア | 目安 |
|--------|------|
| 5 | ポジション取り・ローテが完璧 |
| 3 | 普通 |
| 1 | 無計画・運頼み |

### highlight（見どころ）
| スコア | 目安 |
|--------|------|
| 5 | 切り抜き映えするプレー |
| 3 | 普通のエース |
| 1 | 地味 |

## エントリ追加時のルール

```json
{
  "id": "VIDEO_ID",
  "url": "https://youtu.be/VIDEO_ID",
  "title": "タイトル",
  "map": "マップ名（小文字）",
  "agent": "エージェント名（小文字）",
  "views": 0,
  "score": {
    "smooth": 3,
    "rotation": 3,
    "clutch": 1,
    "highlight": 3
  }
}
```

- IDはアルファベット順を維持
- `views` は初期値 0（YouTube Data API で後から更新予定）
- smooth/rotation/highlight の基準が未定の場合は暫定値 3 を入れる
